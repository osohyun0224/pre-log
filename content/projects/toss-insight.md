## 개요

토스인사이트는 토스의 금융 연구소에서 발행하는 연구 보고서와 회사 소식을 제공하는 웹 서비스입니다. Next.js 15 App Router와 React Query를 활용하여 구축되었으며, 실시간 검색, 페이지네이션, PDF 다운로드 등의 기능을 포함한 리서치 플랫폼입니다.

## 담당 업무

- 토스인사이트 웹 서비스 전체 프론트엔드 개발
- React Query 기반 데이터 페칭 및 캐싱 전략 설계
- 세션 스토리지를 활용한 페이지 간 데이터 전달 최적화
- Suspense + Error Boundary 기반 에러 처리 시스템 구축
- Sentry 연동을 통한 프로덕션 에러 모니터링 구현
- 데스크톱/태블릿/모바일 반응형 대응

## 주요 기능

- 탭 기반 UI (연구 보고서 / 회사 소식) - 데스크톱/모바일 각각 다른 UI
- 보고서 제목 기반 실시간 검색 및 클라이언트 사이드 필터링
- 커스텀 페이지네이션 (현재 페이지 중심 표시, 접근성 고려)
- 세션 스토리지를 활용한 리스트 → 상세 페이지 데이터 전달
- HTML 본문에서 PDF 링크 자동 추출 및 첨부파일 다운로드 배너
- Suspense + Error Boundary 통합 에러 처리 및 Sentry 연동
- URL 파라미터 기반 스크롤 위치 제어 (목록으로 돌아가기)

## 기술적 도전과 해결

### 세션 스토리지를 활용한 API 호출 최소화

**문제:** 보고서 리스트에서 상세 페이지로 이동할 때마다 동일한 데이터를 API로 다시 요청하면 불필요한 네트워크 비용이 발생하고 로딩 지연이 생겼습니다.

**해결:** 리스트에서 항목 클릭 시 해당 보고서 데이터를 세션 스토리지에 저장하고, 상세 페이지에서 세션 스토리지 데이터를 우선 사용하도록 구현했습니다. React Query의 `enabled` 옵션을 활용하여 세션 스토리지 데이터가 있으면 API 쿼리를 비활성화하고, 없을 경우에만 폴백으로 API를 호출하도록 설계했습니다. 사용 후 즉시 세션 스토리지를 삭제하여 메모리를 관리했습니다.

### React Query 캐싱 전략과 클라이언트 사이드 검색

**문제:** 연구 보고서 검색 시 서버에 매번 요청하면 응답 지연이 발생하고, 검색어 입력마다 API를 호출하면 서버 부하가 증가했습니다.

**해결:** 전체 보고서(100개)를 한 번에 가져와 `staleTime: 60초`로 캐싱하고, `useMemo`로 클라이언트에서 정렬과 필터링을 수행하는 방식을 적용했습니다. 검색 시에는 페이지네이션을 비활성화하여 모든 결과를 한 번에 표시하고, 검색어 초기화 시 첫 페이지로 리셋하여 자연스러운 UX를 구현했습니다.

### HTML 본문에서 PDF 링크 자동 추출

**문제:** API에서 제공하는 보고서 본문이 HTML 형식이며, 그 안에 다양한 패턴의 PDF 링크(fr-file 클래스, 일반 a 태그, URL 텍스트, 상대 경로)가 포함되어 있었습니다. 이를 추출하여 별도의 첨부파일 다운로드 배너로 표시하고, 본문에서는 제거해야 했습니다.

**해결:** Positive lookahead를 활용한 다단계 정규표현식 매칭으로 속성 순서에 관계없이 fr-file 클래스의 PDF 링크를 우선 감지하고, 일반 a 태그, URL 텍스트, 상대 경로 순으로 폴백하는 `extractPdfInfo` 함수를 구현했습니다. `removePdfLinksOnly` 함수로 본문에서 PDF 링크만 선택적으로 제거하여 깔끔한 본문 표시와 별도 다운로드 UI를 분리했습니다.

### Suspense + Error Boundary 통합 에러 처리

**문제:** 각 섹션(연구 보고서, 회사 소식)별로 독립적인 로딩/에러 상태를 관리해야 했고, 하나의 섹션 에러가 전체 페이지에 영향을 주지 않아야 했습니다.

**해결:** `@suspensive/react`의 ErrorBoundary와 React Suspense를 통합한 `AppErrorBoundary` 컴포넌트를 구현했습니다. 각 섹션별로 독립적인 에러 바운더리를 적용하여 부분 에러 격리를 달성하고, `onError` 콜백에서 Sentry로 에러를 자동 전송하도록 연동했습니다. React Query의 `throwOnError` 옵션을 섹션별로 다르게 설정하여 유연한 에러 처리 전략을 구현했습니다.

## 성과

- React Query 캐싱과 세션 스토리지 조합으로 불필요한 API 호출 제거 및 페이지 전환 시 즉시 데이터 표시
- 클라이언트 사이드 검색으로 검색어 입력 시 지연 없는 즉시 필터링 구현
- Suspense + Error Boundary 기반 섹션별 독립적 에러 격리 및 Sentry 자동 에러 추적
- 데스크톱/태블릿/모바일 완전 반응형 대응
