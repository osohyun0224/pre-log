## 개요

Vling 디자인 시스템은 서비스 UI 파편화와 중복 개발 문제를 해결하기 위해 구축한 사내 디자인 시스템입니다. Storybook 기반 컴포넌트 문서화, Chromatic 시각 테스트, Changesets 기반 버전 자동화를 도입하고, Figma MCP를 활용한 디자인-개발 생산성 향상까지 총괄했습니다.

## 담당 업무

- 디자인 시스템 구축 및 운영 총괄
- Storybook 기반 컴포넌트 스펙/옵션 문서화 및 플레이그라운드 제공
- Chromatic CI/CD 연동 시각 테스트 및 리뷰 워크플로우 구축
- Changesets 기반 SemVer 버전/변경 이력 자동화 및 배포 파이프라인 구축
- Figma MCP 연동을 통한 디자인-개발 생산성 향상 전략 수립 및 실행

## 주요 기능

- 서비스 전반 UI 가이드라인 및 핵심 컴포넌트 표준화
- Storybook 기반 컴포넌트 문서화 및 디자이너 확인용 플레이그라운드
- PR 단위 Chromatic 시각 테스트 및 리뷰 워크플로우
- Changesets 기반 changelog 생성 → 버전 태깅 → 자동 배포
- Figma MCP를 활용한 협업사 전용 대시보드 빠른 구축

## 기술적 도전과 해결

### 디자인 시스템 구축 및 컴포넌트 운영 체계 확립

**문제:** 서비스 기능이 누적되며 UI가 파편화되고, 팀별로 유사 컴포넌트를 중복 개발하는 문제가 발생했습니다. 디자이너와 개발자가 동일한 UI 기준을 공유하기 어려워 스타일 QA 및 커뮤니케이션 비용이 증가했습니다.

**해결:** 서비스 전반 UI 가이드라인을 수립하고 핵심 컴포넌트를 표준화하여 재사용 가능한 기반을 마련했습니다. Storybook 기반으로 컴포넌트 스펙/옵션을 문서화하고 디자이너가 확인 가능한 플레이그라운드를 제공했습니다. Figma 파일 구조와 코드/스토리 구조를 정렬해 탐색 비용을 줄이고 운영 체계를 고정했습니다.

### 시각 테스트·버전·배포 자동화 구축

**문제:** 컴포넌트 변경 시 UI 회귀 확인이 수동에 의존해 리뷰·QA 리드타임이 길어졌습니다. 변경 이력/버전 관리가 체계화되지 않아 릴리즈 판단이 개인 역량에 의존하고 충돌 위험이 존재했습니다.

**해결:** Chromatic을 CI/CD 파이프라인에 연동해 PR 단위 시각 테스트 및 리뷰 워크플로우를 정착시켰습니다. Changesets 기반 SemVer 버전/변경 이력 자동화를 도입해 릴리즈 규칙을 표준화하고, PR 병합 시 changelog 생성 → 버전 태깅 → 배포까지 자동화하여 배포 프로세스를 개선했습니다.

### Figma MCP 디자인 시스템 연동 및 디자인-개발 생산성 향상

**문제:** 회사 비즈니스 확장으로 삼성전자, 교촌 등 협업사 전용 유튜브 마케팅 대시보드를 빠르게 구축해야 했으나 개발 리소스 병목이 발생했습니다. UI 복잡도는 낮고 재사용 여지가 높아 매번 새로 구성하면 납기 리스크가 커지는 상황이었습니다.

**해결:** 구조적 단순성과 반복성이 높은 대시보드 영역을 자동화 대상으로 정의하고 Figma MCP AI를 전략적으로 도입했습니다. MCP 산출물이 사내 디자인 시스템과 자연스럽게 연결되도록 오토레이아웃·프레임 구조 및 네이밍 규칙을 표준화하고, 색상·타이포·스페이싱을 디자인 토큰 기준으로 정렬했습니다. MCP 산출물을 그대로 사용하지 않고 디자인 시스템 컴포넌트 중심으로 재구성하는 운영 원칙을 수립해 코드 일관성과 유지보수성을 확보했습니다.

## 성과

- 전 서비스 76% 이상 디자인 시스템 기반 개발 완성 및 구성원 간 커뮤니케이션 문서화 구축
- 배포 주기 약 30% 단축 (20분 → 14분), 리뷰 신뢰도 향상 및 릴리즈 의사결정 효율화
- 협업사 전용 대시보드 개발 생산성 40% 절감 (8시간 → 4.8시간)
- Figma MCP 도입 전 2개 협업사 재계약 및 사내 안정적인 수익 모델 확보
